# Clipboard Mage

macOS menu bar app (SwiftUI, macOS 14+): global hotkey (default ⌃⌥⌘C) opens a floating
panel with the current clipboard text; instructions are sent to the Anthropic API and the
transformed result streams back; Enter accepts it into the clipboard.

## Naming — read this first

- **User-facing name:** Clipboard Mage. **Internal names keep "Genie"** — types
  (`GenieSession`, `GenieEngine`, `GenieView`, `GeniePanel`), source dirs
  (`ClipboardGenie/`, `ClipboardGenieTests/`), project/scheme (`ClipboardGenie`),
  and bundle ids (`com.clipboardgenie.app`, `com.clipboardgenie.tests`). This split is
  deliberate (user decision) — do NOT rename either side.
- The test module import is `@testable import Clipboard_Mage` (follows
  `PRODUCT_NAME: "Clipboard Mage"`), not the target name.

## Build / test / run

The `.xcodeproj` is generated by XcodeGen and git-ignored. **After adding, removing, or
renaming any source file you MUST run `xcodegen generate`** before building — XcodeGen
snapshots the file list at generate time.

```sh
xcodegen generate
xcodebuild -project ClipboardGenie.xcodeproj -scheme ClipboardGenie -configuration Debug build
xcodebuild -project ClipboardGenie.xcodeproj -scheme ClipboardGenie test   # full suite
open ~/Library/Developer/Xcode/DerivedData/ClipboardGenie-*/Build/Products/Debug/"Clipboard Mage.app"
pkill -f "Clipboard Mage"   # quit (LSUIElement app — no Dock icon; look for the wand in the menu bar)
```

SourceKit/editor diagnostics ("No such module XCTest", "Cannot find type … in scope")
are chronic false positives here — trust `xcodebuild`, not the editor.

## Architecture (one line each)

- `AppDelegate` (@MainActor) — wiring: hotkey, clipboard watcher, Sparkle updater.
- `PanelController` + `GeniePanel` — borderless floating NSPanel; hides on resign-key.
- `GenieView` — panel UI; Enter submits via `.onSubmit`, Option+Enter = newline.
- `GenieSession` (@MainActor) — state machine. Contains a **generation counter** guarding
  stale in-flight streams and an `inFlightSource` restore-on-cancel; both are load-bearing
  (see `testStaleTransformCannotClobberStateAfterReBegin`, `testCancelMidStreamRestoresSourceText`).
- `GenieEngine` — Anthropic Messages API SSE client (raw URLSession, no SDK).
- `ClipboardService` — pasteboard read/write + 0.5s changeCount polling; suppresses
  its own writes so accepting a result never re-triggers auto-appear.
- `KeychainStore` — API key, service `com.clipboardgenie`, account `anthropic-api-key`.
  Never store the key anywhere else; never log it.

## API gotchas

- Model is exactly `claude-sonnet-5`. The request sends `thinking: {"type": "disabled"}`
  **on purpose** — on Sonnet 5, omitting `thinking` silently enables adaptive thinking
  (slower, costlier). Do not "clean up" that field.
- Never add `temperature`/`top_p`/`top_k` (Sonnet 5 rejects them with 400).
- UserDefaults keys: `autoAppearOnCopy`; hotkey stored by KeyboardShortcuts under `toggleGenie`.
